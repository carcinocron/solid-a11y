## Controlling the Current Tab

By default, the tab components control their own state. One may optionally pass `initialIndex` to
initialize an alternative visible tab panel.

<CodeSampleShowcase source={ProcessedInitialIndexExample} />

It is also possible to fully "control" the tab components by using a `selectedIndex` and `onChange`
pair.

<CodeSampleShowcase source={ProcessedControlledExample} />

## Conditional Content and Classes

`<Tab />` supports optional render callback with a `selected` signal getter on both the `children`
and `classList` props to support conditional classes and tab content.

Moreover, `<TabPanel />` supports an optional callback for `children` (like `<Show />`) for lazy
and/or expensive children.

<CodeSampleShowcase source={ProcessedConditionalExample} />

## Disabled Tabs

Tabs may be disabled so that keyboard interactions ignore these elements.

<CodeSampleShowcase source={ProcessedDisabledTabExample} />

## Vertical Tabs

If vertical tabs are desired, where visually the tabs are oriented from top-to-bottom, rather than
the standard right-to-left, one should set the `orientation` prop as `"vertical"`. This will invert
the keybindings to use up and down arrows properly.
