# Labeling & Descriptions

All `solid-a11y` components have built-in and convenient support for adding ARIA-compliant labels
and descriptions to content. Components utilize native DOM elements by default so one can use the
`aria-labelledby` and `aria-describedby` props directly.

## Using Helpers

With `solid-a11y`, it is more common to use `<Label />` or `<Description />` which will
automatically annotate associated DOM elements using Solid's context API.

Note in the following example, one does not need the boilerplate wrapping code of `<LabelGroup />`
and `<DescriptionGroup />`. Most components in `solid-a11y` (`<RadioGroupOption />` in this case)
already wrap their children by default with these labeling contexts, so there's no need to do this
by hand.

## Visually Hidden Elements

One may want to add context or labeling only for screen reader users without affecting the visual
content on the page. In such cases, `display: none;` is not appropriate since screen readers will
not speak this content. For this, one should use something like the [`.sr-only` utility class in
Tailwind CSS][tailwind_sr_only], which makes the text visually hidden without removing it from the
